Life Saver

Life Saver keeps a running backup of your most important MODO preferences.

Quickstart

  1. Install the kit
  2. When MODO starts for the first time, select the config fragments to back up. In most cases, it's best to leave all of these enabled. See Preferences below for a description of each fragment.
  3. Once you have configured your MODO experience, restart MODO. The MODO config will be saved as soon as you successfully quit MODO, and Life Saver will create its first backup the next time MODO starts.
  4. Use MODO as normal.

Once Life Saver is up and running, it will keep a running backup of your most important preferences in its own Configs folder. When problems arise that force you to use System > Reset Preferences, your important settings will not be affected.

Preferences

Enable/Disable config fragments - When enabled, the following config fragments will be extracted from the MODO config file each time MODO starts. When disabled, any existing backup file will be deleted, and from then on the config fragment will not be backed up.

  • Input Remapping - Hotkeys and mouse behaviors
  • Preset Browser Paths - Any custom folder locations added to the F6 preset browser
  • Preferences - Most of the settings stored under System > Preferences
  • Global Settings - Certain settings stored under System > Preferences
  • User Values - Mainly used by third-party kits and plug-ins to store settings between sessions
  • Recent Files and Folders - The files listed under File > Open Recent..., as well as File > Open dialog paths, etc.
  • Check for Updates - Whether or not to automatically check for MODO updates
  • Post Usage Stats - Whether or not to send anonymous usage statistics to Foundry

Reset MODO Configs - Equivalent to System > Reset Preferences. Deletes your main MODO config, leaving all Life Saver backups in place.

Reset Backup Configs - Deletes Life Saver backup configs, but leaves the main MODO config file untouched. (Rarely used.)

Reset All Configs (factory reset) - Deletes the main MODO config file AND all Life Saver backup configs, thus reverting MODO to its factory defaults.

Technical Details

When MODO quits successfully, it saves its configuration settings from RAM to the main MODO config. This config file includes literally everything about your MODO session, so that the next time MODO starts, it will look exactly the way you left it.

When problems arise, they are often stored into the main MODO config file. Corrupted MODO config files can lead to crashes, erratic behavior, and all sorts of unexpected glitches. Many MODO problems can be fixed by deleting the main MODO config.

Unfortunately, this means deleting all of your configs, including the ones you spent time carefully setting up: your custom hotkeys, units and document settings, Preset Browser locations, et cetera. Until now, a preferences reset meant resetting everything.

Configs like Input Remapping and Preset Browser paths are almost never corrupted. They are simple, straightforward, and there is almost never a reason to reset them.

Every time MODO starts up, Life Saver runs a script that crawls over the entire MODO config file looking for certain fragments. If it finds something worth keeping, it merges it into a backup config file in the kit's Configs folder. (Each fragment is stored inside its own backup file.)

With Life Saver installed, the extracted fragments are kept safe, so that running System > Reset Preferences won't delete them. Since these fragments of the kind that are almost never corrupt, you should be able to keep them around without a problem. The next time a MODO config is corrupted, just run System > Reset Preferences, and the corrupt stuff will be deleted, leaving the backups in place.

If, however, you want to delete your backups, you can always (a) uninstall the kit, or (b) run either Reset Backup Configs or Reset All Configs to clean house. These scripts simply delete the backup files from the kit's Configs folder.

Life Saver is, in effect, a hack. If the MODO config system were more robust, we wouldn't have to do this. I hope that someday it will be unnecessary. For now, I hope it's helpful. Let me know if I can make it better.
